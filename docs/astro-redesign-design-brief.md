# Hippocampus's Garden Astro Redesign Design Brief

## 目的

このドキュメントは、`https://hippocampus-garden.com/` の Astro リビルドに向けたデザイン方針書である。

目標は次の 3 点。

1. 2026 年の基準で見ても古びない、静かで知的な個人ブログにする。
2. コンテンツの強さを UI がきちんと増幅する状態にする。
3. 「珊瑚礁」というコンセプトを、配色・余白・タイポグラフィ・画像・動きに一貫して反映する。


## デザインの結論

現行サイトは「海っぽい色を使った Gatsby ブログ」であり、「珊瑚礁をモチーフにした Shion Honda の知的なブログ」にはまだなっていない。

Astro リビルドでは、次の方向に振る。

- トーン: 静かで知的
- 印象: 端正、落ち着いている、信頼感がある、少しだけ有機的
- 避けるもの: 派手さ、スタートアップ風の強い CTA、ガラス UI の多用、装飾過多、情報過多
- 目指すもの: 読みやすさ、余白の質、精度の高いタイポグラフィ、抑制されたブランド表現


## ブランドコンセプト

### コアメタファー

ブログは庭ではなく珊瑚礁である。

珊瑚礁は以下の性質を持つ。

- 長い時間をかけて形成される
- 多様な知識や関心が共存する
- 一見静かだが、内部には豊かな生態系がある
- 表面的な派手さではなく、近づくほど構造の美しさが見える

このメタファーを UI に翻訳すると、次のようになる。

- 余白は広めに取る
- 配色は深海系をベースにし、珊瑚色はアクセントに限定する
- 装飾は面ではなく、層や流れとして扱う
- コンテンツの階層は、珊瑚礁の「群生」のように整然と配置する

### 色の使い方

- ページ全体は白基調でよい。ただし純白ベタではなく、ごく薄い水色や青灰を背景に混ぜる。
- ブランド色はヘッダー全面塗りではなく、局所的に使う。
- `reef-coral` はリンク hover、タグ active、注目箇所などのアクセント用途に限定する。
- 記事本文は高コントラストを維持し、ブランド色を本文可読性より優先しない。


## タイポグラフィ方針

### 基本方針

現行のように英語本文、日本語本文、UI ラベルをほぼ同じ調子で処理しない。

最低でも次の 3 系統に分ける。

- 日本語本文
- 英語本文
- UI ラベル、ナビゲーション、メタ情報

### 方向性

- 本文: 長文読書に強い書体を使う
- UI: 視認性の高い sans-serif を使う
- 見出し: 本文と同系統でもよいが、ウェイトと字間で知性を出す

### 要件

- 日本語と英語が混在しても破綻しないこと
- コードブロック、数式、脚注、引用でリズムが崩れないこと
- 記事タイトル、日付、タグ、補助説明文の階層差が明確であること
- 1 行長は読みやすさを優先し、本文カラムを広げすぎないこと


## 情報設計方針

### 全体原則

- ホームは「最近書いたものを読む場所」として明快にする
- 記事ページは「読むこと」に集中させる
- About は「著者の現在地と強みを短時間で理解できる」構成にする
- タグや人気記事は補助情報として扱い、主導権を持たせない

### ホーム

ホームの主役は `Recent posts` とする。

採用する構成:

1. Recent posts にすぐ接続する最小限の導入
2. Recent posts
3. Topic / Tags
4. Ranking modules
5. Short author block

採用しない構成:

- Featured essay
- ランキングを主役に据える構成
- 右カラムに大量の補助情報を積む古いブログレイアウト

### 記事詳細

記事ページは本文中心にする。

優先順位:

1. タイトル
2. 日付、読了時間、タグ
3. 本文
4. 目次
5. 関連記事
6. 著者情報

TOC、関連記事、タグ群は本文を邪魔しない配置にする。常設サイドバー前提ではなく、画面幅に応じて自然に再配置される構成が望ましい。

### About

About は履歴書ではなく、現在のあなたの専門性を伝えるページにする。

必要なセクション案:

1. One-line introduction
2. What I work on
3. Selected writing
4. Career snapshot
5. Books / Talks / External writing
6. Links


## 画面別デザイン指針

### 1. Home

目的:

- 最新記事に自然に入ってもらう
- ブログのテーマと著者の専門領域を短時間で理解してもらう

要件:

- ヘッダーはブランドを感じるが、圧が強すぎない
- ファーストビューで最新記事一覧にすぐアクセスできる
- 記事カードは一覧性を保ちつつ、各記事の個性も見える
- 日本語・英語記事が混在しても美しく見える
- タグ一覧は「索引」として扱い、ノイズにしない

構成案:

- Top bar / site identity
- Optional short intro line
- Recent posts grid or stack
- Topic cloud or curated tag section
- Compact ranking section such as Trending / Most read
- Minimal author block

補足:

- Home の導入は必須ではない
- 一文の導入がノイズになるなら置かず、ヘッダー直下から `Recent posts` を始めてよい
- Home の主役は常に記事一覧であり、視線を奪うヒーロー表現は採用しない

ランキングの扱い:

- Ranking は入れてよい
- ただし `Recent posts` より大きく見せない
- 見せ方は「補助的な発見導線」とし、主導権は持たせない
- カードの連打よりも、コンパクトで静かなリスト表現を優先する

### 2. Post

目的:

- 長文を快適に読めること
- 数式、コード、画像、脚注を含んでも一貫した品位を保つこと

要件:

- 本文カラムは読みやすい幅に固定する
- 見出しの階層差を明確にする
- コードブロックは技術記事に相応しい視認性にする
- 数式と本文の余白ルールを整える
- 記事末のナビゲーションは簡潔にする
- 共有導線が必要なら控えめに置く

構成案:

- Article header
- Optional TOC
- Body
- Notes / Footnotes styling
- Related posts
- Author note

### 3. About

目的:

- 著者の信頼感と専門性を伝える
- 初めて来た人が「何を書いている人か」を理解できるようにする

要件:

- 顔写真、肩書き、拠点、関心領域をコンパクトに見せる
- SNS アイコン羅列だけで終わらせない
- 外部リンクは整理し、優先度を付ける
- 英語版 / 日本語版の切り替えをわかりやすくする


## ビジュアルシステム

### 背景

- ページ背景は単色ではなく、ごく薄い海流のような勾配を使う
- ノイズ感の強いテクスチャは使わない
- 珊瑚礁らしさは背景全面よりも、セクションの境目やアクセントで示す

### コンテナ

- 白いカードを無数に並べるのは避ける
- セクション単位の panel は使ってよいが、影より境界線とトーン差で見せる
- 境界は `line` 系の淡い色で処理する

### モーション

- 初期リリースでは、アニメーションは基本的に使わない前提で設計する
- どうしても必要な場合でも、読書体験を邪魔しない最小限にとどめる
- `prefers-reduced-motion` を必ず尊重する


## コンポーネント方針

### Header

- 現行の全面グラデーション帯は廃止する
- サイト名、短い説明、主要ナビだけに絞る
- ブランド感は色ベタではなく、余白、線、タイポ、微細な背景表現で出す

### Post Card

- タイトル、説明、日付、タグの階層を明快にする
- サムネイルは optional にしてよい
- サムネイルを使う場合は比率を統一し、質の低い画像で一覧が崩れないようにする
- 長いタイトルや日本語タイトルで崩れないこと

### Tag

- タグ数を大量に露出しすぎない
- UI 上は index として静かに見せる
- カラーアクセントを使うなら selected / hover のみ

### Footer

- 「Built with Gatsby」は削除する
- フッターは補助ナビと著作情報だけで十分
- 主張しすぎないが、雑にも見えない処理にする


## UX / アクセシビリティ原則

- キーボード操作だけで主要導線に到達できること
- Skip link を入れること
- フォーカスリングを消さないこと
- 日本語・英語の混在で読み上げや見出し構造が破綻しないこと
- アイコンだけのリンクには必ずラベルを付けること
- 画像 alt を generic にしないこと
- モバイルでも本文、コード、表が破綻しないこと
- ダークモードは初期リリース必須ではないが、トークン設計は対応可能にしておくこと


## コンテンツ戦略に関する前提

- 更新頻度は高くない
- そのため、ホームは運用コストが高い手動キュレーションを避ける
- Featured essay は置かない
- 最新記事が自然に価値を伝えられる設計にする
- 重要なのは「更新頻度を演出すること」ではなく、「蓄積された知的資産を上品に見せること」


## Astro 実装方針

### 技術方針

- コンテンツは Markdown or MDX ベースを維持する
- レイアウトは Astro コンポーネント主体にする
- インタラクションは必要箇所だけ islands 化する
- まずは静的生成を前提にし、読書体験を最優先する

### デザイントークン

最初に定義すべきもの:

- colors
- typography
- spacing
- radii
- border
- shadow
- content widths
- motion durations

### 必須レイアウト

- `BaseLayout`
- `HomeLayout`
- `PostLayout`
- `AboutLayout`

### 先に整えるべき共通部品

- Header
- Footer
- PostCard
- Tag
- Prose styles
- TOC
- Pagination


## 初期スコープ

リビルドの第 1 段階では、以下を完成対象にする。

1. Home
2. Post page
3. About
4. Tag index
5. Global tokens and typography

第 1 段階では後回しでよいもの:

- 高度な検索
- アニメーション実装全般
- ダークモード
- 記事の手動ピン留め
- 複雑なランキング UI


## デザインレビュー観点

実装レビューでは、次を毎回確認する。

- 珊瑚礁コンセプトが「色名」ではなく UI に落ちているか
- 静かで知的なトーンが守られているか
- 本文の読みやすさが最優先されているか
- 情報の主従が明確か
- 補助情報が主役を食っていないか
- 日本語と英語の両方で美しいか
- 運用コストが低いか


## 参考となる最終イメージ

目指すべき完成像は以下。

- 第一印象は静かで洗練されている
- 数秒で「機械学習、ソフトウェア、読書、知的な個人ブログ」と伝わる
- 読み始めると本文が非常に読みやすい
- ブランドは強く主張しないが、記憶に残る
- 2020 年の個人技術ブログではなく、2026 年のプロフェッショナルな個人出版物に見える
