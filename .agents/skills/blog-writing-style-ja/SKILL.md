# 日本語ブログを書く

著者の考え方、具体性、慎重さを残したまま、論旨と読みやすさを整える。

## 執筆前に文体を合わせる

1. 対象の草稿を最後まで読む。
2. `astro/src/content/posts/` から、同じ言語・ジャンルの記事を2〜4本読む。原則として日付が新しいものを優先し、構成が近ければ過去記事も参照する。
3. 未公開記事や移行後の内容についてはリポジトリを正とする。公開中の見え方に合わせる依頼の場合だけ本番サイトも確認する。
4. 特徴的な一文を模倣するのではなく、導入、論の進め方、確信度、段落のリズムを抽出する。

ジャンル別の参照候補:

- キャリア・考察: `commodity_knowledge`, `dual_career_practice`, `ai_and_employment`
- 技術・研究: `ai_papers_2025_ja`, `tuning_playbook`
- 読書・書評: `books_2025`, `book_review_patterson`
- 生活・家族: `bottle_refusal`, `paris_tips`

## 文体を合わせる

- 落ち着いた説明口調を使い、煽らない。
- 体験、観察、以前の記事、または具体的な問題意識から自然に書き始める。
- 背景や前提を共有してから主張し、具体例や比較で支える。
- 主観は「〜と思います」「〜と感じました」などで明示する。ただし、すべての文を弱めない。
- 事実、解釈、推測を区別し、不確実性や一般化の限界を残す。
- 読者に不要な一般論より、固有名詞、数値、場面、行動を書く。
- 技術用語は必要に応じて英語を併記し、初出で説明する。
- 一つの段落では一つの役割を担わせ、論点が変わる場所で段落を切る。
- 列挙、表、脚注は情報整理に役立つ場合に使う。

## 文体を壊さず推敲する

- 実体験、控えめなユーモア、著者固有の例、慎重な言い回しを残す。
- 長い一文は主語と述語を確認し、必要なら分割する。
- 重複、曖昧な指示語、同義の言い換えの連続を削る。
- 「重要です」「示唆的です」で止めず、何が、なぜ、どの範囲でそう言えるのかを書く。
- 生成AIにありがちな抽象的な導入、意味のない接続表現、不自然に均整の取れた構成、同じ主張の言い直し、必要以上に大げさな結論、情報を加えない流暢なだけの文を避ける。具体的な事実、経験、例、因果関係を優先する。
- 文章を印象的に見せるためだけの装飾的・長大な比喩を追加しない。比喩は複雑な仕組みを理解しやすくする場合に限って短く使い、複数の比喩を重ねたり、著者の意図を超えて展開したりしない。
- 「昨今」「急速に変化する時代において」のような中身の薄い導入や、結論の言い直しだけのまとめを加えない。
- 根拠のない事実、引用、体験、確信を補わない。
- `## はじめに`、`## おわりに`、定型的なまとめを機械的に追加しない。
- Markdown、脚注、リンク、HTML、コード、数式は、構成変更の依頼がない限り保つ。

## 仕上げを確認する

- 導入から「なぜ今この話を書くのか」が伝わるか確認する。
- 各節が新しい論点、根拠、具体例のいずれかを加えているか確認する。
- 著者の経験と一般的な助言が混同されていないか確認する。
- 不自然に均整の取れた段落、同じ接続詞の反復、生成AIらしい抽象表現を削る。
